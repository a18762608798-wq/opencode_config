# pandas apply 的使用时机与性能优化

**日期**: 2026-07-19 20:42
**分类**: 编程/Python
**标签**: #pandas #apply #性能优化 #向量化

## 背景

用户在学习 pandas 分组操作后，进一步询问 `apply` 的使用时机和性能考量。

## 内容

### apply 的本质

`apply` 本质上是 Python 层面的逐行/逐组循环，没有利用 pandas 的向量化加速，因此速度较慢。

### 使用优先级

1. **优先用内置方法**：`.sum()`, `.mean()`, `.std()`, `.count()` 等已高度优化的 C 层面实现
2. **其次用 agg**：一个操作完成多个聚合，减少遍历次数
3. **万不得已用 apply**：仅当操作逻辑极其特殊、无法用内置方法或向量化步骤组合实现时才使用

### 典型案例与优化

**不好的写法（慢）**：
```python
df.groupby('key').apply(lambda g: g['a'].sum() / g['b'].mean())
```

**优化后（快）**：
```python
sum_a = df.groupby('key')['a'].sum()
mean_b = df.groupby('key')['b'].mean()
result = sum_a / mean_b
```

将 apply 中的复杂逻辑拆解为多个向量化聚合步骤再合并，避免 Python 循环开销。

### 何时可以用 apply

- 操作确实无法拆解为向量化步骤
- 数据量较小，性能不是瓶颈
- 原型开发阶段，先快速验证逻辑

## 要点

- `apply` 是 Python 层循环，没有向量化，速度慢
- 优先使用内置聚合方法（`.sum()`, `.mean()` 等）和 `agg`
- 可拆解的 apply 操作应重写为多个向量化步骤的组合
- 仅在逻辑特殊且无法向量化时使用 apply
