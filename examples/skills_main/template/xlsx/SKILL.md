# XLSX 创建、编辑和分析

| 任务 | 方法 |
|---|---|
| **创建**或**编辑**带公式/格式的内容 | `openpyxl` — 见下方注意事项 |
| **批量数据**导入或导出 | `pandas`（`read_excel`、`to_excel`） |
| **快速查看**工作表 | `markitdown file.xlsx` — 每个工作表以 `## SheetName` 显示；也支持 `.xlsm`。无单元格坐标，不要据此规划编辑 |
| **读取**模型（公式*和*值） | 两次 `load_workbook` — 见下方注意事项 |

> `openpyxl`、`pandas` 和 `markitdown` 已预装——不要先运行 `pip install`；直接编写脚本并导入。仅当导入失败（或 `markitdown` 命令缺失）时：`pip install` 缺失的包。

> 以下脚本路径相对于此技能的目录。

## 每个输出的要求

- **全程使用专业字体**（Arial、Times New Roman），除非用户另有要求。
- **零公式错误。** 当 `recalc.py` 报告 `errors_found` 时绝不要交付。如果你认为错误在你之前就存在，请证明：使用 `data_only=True` 加载*原始*文件并查看该单元格。你引入的错误和你继承的错误看起来完全一样。
- **使用公式，绝不硬编码结果。** 写 `sheet['B10'] = '=SUM(B2:B9)'`，而不是 Python 计算的总和。工作表必须在其输入变化时重新计算。
- **严格遵循用户的需求说明。** 精确的表名称、精确的列标题和他们指定的公式。无论多么优雅，计算其他内容的重新设计都是失败。
- **记录每个假设和硬编码数字**在读者能看到的地方——单元格注释或表格末尾的相邻单元格。尽可能引用真实来源（`来源：公司 10-K，2024 财年，第 45 页，收入注释，[SEC EDGAR URL]`）；当数字来自用户时，直接说明。
- **你为他人填写而创建的工作簿**需要简短的图例，说明哪些单元格可编辑，以及一行展示预期格式的真实示例值。绝不要将此类行添加到需要编辑的文件中。
- **编辑现有文件：完全遵循其约定。** 它们的优先级高于此处每条指南。首先找到其指定的输入单元格——不同的字体颜色、填充或阴影标记——只在那些位置写入，并保持每个现有公式不变。

## 重新计算（每当文件包含公式时必须进行）

openpyxl 将公式写入字符串，**不带缓存值**。在重新计算之前，每个公式单元格对任何读取缓存值的内容（`pandas`、`load_workbook(data_only=True)` 和大多数预览器）都显示为 `None`。

```bash
python scripts/recalc.py output.xlsx [timeout_seconds]   # 默认 30
```

LibreOffice 计算每个公式，文件被**原地重写**，并返回 JSON：
`status`（`success` | `errors_found`）、`total_formulas`、`total_errors` 以及
`error_summary`，每种错误类型最多列出 100 个单元格（`locations_truncated` 说明省略了多少——信任 `total_errors`，而非列表长度）。修复它指出的内容并再次运行。**带有 `error` 键而不是 `status` 的 JSON 意味着没有任何内容被重新计算**，只有这种情况以非零退出——`errors_found` 以 0 退出，所以绝不要将干净的退出视为干净的工作簿。

**绿色的 recalc 证明你的公式可以*求值*，而不是它们*正确*。** 差一的范围或引用错误行会产生干净、无错误但数字错误的文件。在构建网格之前，先写 2-3 个公式并检查它们是否拉取了你期望的值。

**链接到另一个文件的工作簿**，如果使用 openpyxl 重新保存然后重新计算，会丢失这些链接。此类公式显示为 `='[1]Returns Analysis'!$B$2`——其中 `[1]` 是工作簿外部引用列表的索引，指向的是*磁盘上的另一个文件*，而非工作表。该文件通常不在此处，因此单元格的缓存值是唯一保存其数据的东西。openpyxl 在保存时会剥离该值；然后 LibreOffice 必须真正解析引用，失败后写入 `#NAME?`，并删除每个链接。`recalc.py` 在此状态下拒绝运行——在保存覆盖之前从原始文件复制这些单元格的值（`--force` 可覆盖，并接受损失）。

## 选择能通过验证的公式

LibreOffice 实现的函数比 Excel 少，它无法求值的函数会变成你交付文件中的字面 `#NAME?`。

- **优先使用 Excel 2007 时代的函数**——`SUMIFS`、`INDEX`、`MATCH`、`IFERROR`、`SUMPRODUCT`——这些不需要前缀。
- **六个 2007 年后的函数可用，但需要 `_xlfn.` 前缀**，因为 openpyxl 将你的公式原样写入 XML，而 Excel 使用带前缀的方式存储 2007 年后的名称（其 UI 隐藏前缀）：`_xlfn.TEXTJOIN`、`_xlfn.CONCAT`、`_xlfn.IFS`、`_xlfn.SWITCH`、`_xlfn.MAXIFS`、`_xlfn.MINIFS`。不加前缀写入时，每个都会显示 `#NAME?`。
- **绝不使用 `XLOOKUP`、`XMATCH`、`SORT`、`FILTER`、`UNIQUE` 或 `SEQUENCE`。** 运行时的 LibreOffice 在任何前缀下都无法求值。较新的构建确实支持求值，但它们是溢出数组函数，而 openpyxl 写入的文件没有溢出元数据，因此只有区域的左上角单元格能得到值——而 `recalc.py` 在截断的结果上报告 `total_errors: 0`。使用 `INDEX`/`MATCH` 做查找，并在写入单元格之前在 Python 中进行排序、筛选和去重。
- LibreOffice 无法解析的公式会被写回**小写**形式——这是 `#NAME?` 旁边的一个快速识别标志。

## openpyxl 注意事项

- **读取模型需要两次加载。** `data_only=True` 产生缓存值但公式丢失；默认产生公式字符串但没有值。一次加载不能同时获取两者。
- **`data_only=True` 如果保存是破坏性的。** 该工作簿没有公式留存，因此保存会将每个公式替换为字面值——永久性的。
- **`data_only=True` 在 openpyxl 刚写入的文件上返回 `None`**——先运行 `recalc.py`。（结果为 `""` 的公式也显示为 `None`。）
- **合并单元格：只写左上角锚点。** 区域中的每个其他单元格都是 `MergedCell`，其 `.value` 是只读的。
- **`.xlsm` 会丢失其宏**，除非向 `load_workbook` 传递 `keep_vba=True`。
- **包含空格的表名称在跨表引用中必须加引号**：`='Assumptions Inputs'!$B$5`。不加引号会求值为 `#VALUE!`。

## 财务模型

除非用户另有说明，或现有文件已采用其他方式。

**颜色：** 蓝色文字（`0,0,255`）用于硬编码输入和情景杠杆 · 黑色用于公式 ·
绿色（`0,128,0`）用于链接到另一个工作表 · 红色（`255,0,0`）用于链接到另一个文件 ·
黄色填充（`255,255,0`）用于关键假设和用户应填写的单元格。

**数字：** 货币 `$#,##0`，单位在表头中注明（`收入 ($mm)`）· 零
显示为 `-`，包括百分比（`$#,##0;($#,##0);-`）· 负数用括号 ·
百分比 `0.0%`，**存储为小数**（`0.15` 显示为 `15.0%`；存储 `15` 显示为
`1500.0%`）· 估值倍数 `0.0x` · 年份为文本（`"2024"`，绝不为 `2,024`）。

**结构：** 每个假设放在自己的标记单元格中，由使用它的公式引用
（`=B5*(1+$B$6)`，绝不为 `=B5*1.05`）· 公式在每期投影中保持一致，因为
中间单独编辑的单元格是最常见的静默错误 · 保护可能为零的分母。
