# TUI 基础操作

## 认识 TUI 界面

OpenCode 的 TUI（Text User Interface）是主要交互界面。启动后你会看到：

```
┌─ OpenCode ────────────────────────────────────────────────┐
│  Agent: Build ◄─── 当前 Agent                  会话时间   │
│  Model: claude-sonnet-4 ◄─── 当前模型          花费: $0.05│
│───────────────────────────────────────────────────────────│
│                                                           │
│  [User]                                                   │
│  > 创建一个 Python 脚本来分析日志文件                      │
│                                                           │
│  [Assistant]                                              │
│  我来帮你创建一个日志分析脚本...                           │
│                                                           │
│───────────────────────────────────────────────────────────│
│  > _ ◄─── 输入区域                                        │
│                                                           │
│  Ctrl+T 模型  Tab 模式  Ctrl+C 中断  / 命令  Esc 退出    │
└───────────────────────────────────────────────────────────┘
```

### 界面区域

| 区域 | 说明 |
|------|------|
| 顶部栏 | 当前 Agent、模型、会话信息 |
| 对话区 | 你和 AI 的对话历史 |
| 输入区 | 输入指令的地方，支持多行 |
| 底部栏 | 快捷键提示 |

## 核心操作

### 文件引用 (@)

`@` 符号是使用频率最高的功能。它可以：

- 🔍 **模糊搜索文件**：输入 `@auth` 会列出所有包含 "auth" 的文件
- 📎 **引用文件到对话**：选中文件后，其内容会被纳入上下文

```
@src/auth/login.ts 帮我分析这个登录模块的安全问题
```

多个文件引用：

```
对比 @src/auth/login.ts 和 @src/auth/signup.ts 的密码处理逻辑
```

### 斜杠命令

在输入框中输入 `/` 会弹出命令面板：

| 命令 | 说明 | 示例 |
|------|------|------|
| `/help` | 显示全部命令 | `/help` |
| `/init` | 初始化 AGENTS.md | `/init` |
| `/connect` | 配置 LLM 提供商 | `/connect` |
| `/model` | 切换模型 | `/model` |
| `/agent` | 切换 Agent | `/agent` |
| `/sessions` | 查看/切换会话 | `/sessions` |
| `/new` | 新建会话 | `/new` |
| `/timeline` | 查看会话时间线 | `/timeline` |
| `/compact` | 压缩会话上下文 | `/compact` |
| `/export` | 导出对话 | `/export` |
| `/undo` | 撤销上次修改 | `/undo` |
| `/redo` | 重做被撤销的修改 | `/redo` |
| `/clear` | 清空对话（同 `/new`） | `/clear` |
| `/cost` | 查看花费 | `/cost` |
| `/diff` | 查看当前修改 | `/diff` |
| `/status` | 查看会话状态 | `/status` |

| 快捷键 | 功能 |

| `Ctrl+P` | 打开命令面板 |

## Plan 模式（规划模式）

Plan 模式的特点是 **只读不写**。AI 会：

- ✅ 阅读和分析代码
- ✅ 提出方案和建议
- ✅ 回答技术问题
- ❌ 不会修改任何文件
- ❌ 不会执行写入操作

### 何时使用 Plan 模式？

```
# 理解代码
解释一下这个中间件的执行顺序

# 方案设计
我想给这个 API 加缓存，有哪些方案？各自的优缺点？

# 代码审查
审查 @src/api/handlers.ts 的安全问题

# 需求分析
我需要实现一个消息队列，帮我分析是选 Redis 还是 RabbitMQ
```

## Build 模式（构建模式）

Build 模式是 **完全读写** 的。AI 会：

- ✅ 创建、修改、删除文件
- ✅ 执行命令行操作
- ✅ 安装依赖
- ✅ 运行测试

### 何时使用 Build 模式？

```
# 功能开发
在 @src/routes 下新建 users.ts，实现 CRUD 接口

# Bug 修复
@src/utils/parser.ts 第 42 行的正则表达式有问题，修复它

# 重构
把所有 API 调用的错误处理统一改成 try/catch 模式
```

> ⚠️ **注意**：Build 模式下 AI 会直接修改你的文件。建议：
>
> - 操作前确保代码已提交 Git
> - 使用 `/diff` 查看修改内容
> - 不满意可以 `/undo` 撤销
