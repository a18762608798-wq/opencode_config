# Plan 模式

## 什么是 Plan 模式？

Plan 模式是 OpenCode 的**规划与分析模式**。在这个模式下，AI 只会**阅读和理解**代码，**不会修改任何文件**。

这是 OpenCode 区别于其他 AI 编程工具的重要特性——它把「思考」和「行动」分开了。

## Plan 模式的使用场景

### 1. 代码理解

```
# 快速理解陌生代码库
解释一下这个项目的整体架构和数据流

# 深入某个模块
@src/services/auth.ts 这个认证模块的处理流程是怎样的？
```

### 2. 方案设计

```
# 架构决策
我想给系统加一个消息队列，RabbitMQ 和 Kafka 怎么选？
结合我们的项目特点给个建议

# 技术选型
前端状态管理用 Zustand 还是 Redux Toolkit？
分析一下在我们这个项目规模下的优劣
```

### 3. 需求分析

```
# 拆解需求
用户想要一个"收藏夹"功能，
帮我梳理一下需要哪些接口、数据库怎么设计、前端需要哪些组件
```

### 4. 代码审查

```
# 安全审查
审查 @src/api/payment.ts 的安全问题

# 性能分析
@src/utils/data-processor.ts 这个函数处理 10 万条数据需要多久？
有没有优化空间？
```

### 5. 生成计划

```
# 制定开发计划
我要重构这个项目的错误处理机制，
先帮我分析现状，然后给出一个分步骤的重构计划
```

## Plan 模式的输出格式

Plan 模式下，AI 通常会给出结构化输出：

````markdown
## 📋 分析结果

### 当前状态
- 发现问题：3 处
- 优化建议：5 处

### 🔴 需要修复
1. [安全问题] `login.ts:42` — SQL 注入风险
2. [性能问题] `parser.ts:156` — O(n²) 复杂度

### 🟡 建议优化
1. 统一错误处理模式
2. 添加请求频率限制
3. ...

### 📝 实施计划
1. 第一步：修复安全问题（预计 30 分钟）
2. 第二步：重构错误处理（预计 2 小时）
3. ...
````

## Plan → Build 工作流

推荐的开发流程：

```
1. Plan 模式：分析需求
   "我想实现用户积分系统，帮我分析一下"

2. 审查 Plan
   "这个方案里，积分过期逻辑放在哪里？"

3. 调整 Plan
   "把积分过期改成定时任务处理"

4. 切换到 Build（按 Tab）
   确认计划无误后开始实现

5. Build 模式：逐步实现
   "按照刚才的计划，先创建数据库迁移文件"
```

## 在 Plan 模式中引用上下文

### 引用多个文件

```
对比 @src/auth/login.ts @src/auth/signup.ts @src/auth/reset.ts
这三个模块的错误处理方式，给出一致性建议
```

### 引用目录

```
@src/api/ 分析这个目录下的 API 设计是否遵循 RESTful 规范
```

### 引用 Git 历史

```
查看最近的提交记录，分析这个项目的开发节奏和代码质量趋势
```

## Plan 模式的技巧

### 1. 给出具体约束

```
# ❌ 太模糊
这个项目怎么优化？

# ✅ 具体明确
在不改变外部 API 的前提下，
如何优化 @src/services/search.ts 的搜索速度？
预算：最多增加 50MB 内存使用
```

### 2. 提供决策标准

```
选择缓存方案，按以下优先级评估：
1. 实现复杂度（越低越好）
2. 命中率（越高越好）
3. 运维成本（越低越好）
```

### 3. 要求多方案对比

```
给我三个用户认证方案，每个方案列出：
- 优点
- 缺点
- 适用场景
- 实现难度（1-10）
- 安全等级（1-10）
```

### 4. 迭代式深入

```
第一步：整体架构分析
→ "分析项目结构"

第二步：聚焦模块
→ "@src/api/ 的中间件链是怎么组织的？"

第三步：具体实现
→ "这个限流中间件的算法是什么？
   有没有改进空间？"
```

## 最佳实践

1. 🧠 **先 Plan 后 Build**：复杂任务先在 Plan 模式想清楚
2. 📝 **保存 Plan 结果**：用 `/export` 导出分析结果作为文档
3. 🔍 **充分利用只读安全**：Plan 模式可以大胆探索代码库
4. 🎯 **明确边界**：告诉 AI 哪些是约束条件，哪些可以灵活处理
5. ⚡ **快速切换**：用 Tab 键在 Plan/Build 间快速切换
